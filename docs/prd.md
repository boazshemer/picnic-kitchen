# PRD - Product Requirements Document
## אפליקציית ניהול משימות למפעל מזון

### מטרה
אפליקציית Web לניהול משימות יומיות במפעל מזון, המאפשרת מעקב אחר הכנת מנות על ידי צוות המטבח.

### משתמשים
- **שפית ראשית**: מזינה את רשימת המנות היומית למערכת
- **16 טבחים**: מבצעים את הכנת המנות
- **סה"כ**: ~400 מנות ביום

### תהליך עבודה
1. השפית מזינה למערכת את רשימת המנות הנדרשות ליום
2. כל מנה משויכת אוטומטית לטבח ברירת מחדל (בהתאם לסוג המנה)
3. הטבחים יכולים לצפות במשימות שלהם ולעדכן סטטוס
4. המערכת שולחת עדכונים לשרת חיצוני (של גיא) בזמן אמת

### אינטגרציה
- **שרת חיצוני של גיא**: המערכת תשלח עדכוני סטטוס והזמנות לשרת חיצוני באמצעות API

### דרישות טכניות
- ממשק Web מודרני וידידותי למשתמש
- אחסון נתונים ב-Supabase
- Backend ב-Python (FastAPI)
- תמיכה בעברית מלאה
- עדכונים בזמן אמת

### מטרות העסקיות
- שיפור יעילות הכנת המנות
- מעקב מדויק אחר ביצוע משימות
- צמצום טעויות בהזמנות
- שקיפות מלאה בתהליך הייצור
